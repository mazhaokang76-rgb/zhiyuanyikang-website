# 智缘益慷网站 - 详细部署指南

## 第一步：上传代码到GitHub

### 1.1 准备工作
1. 登录您的GitHub账号：https://github.com/mazhaokang76-rgb
2. 确保您有仓库的管理权限

### 1.2 上传方法
有多种方法上传代码到GitHub：

#### 方法一：通过GitHub网页界面上传
1. 访问您的仓库：https://github.com/mazhaokang76-rgb
2. 点击 "uploading an existing file" 或直接拖拽文件
3. 将整个 `/workspace/GitHub部署包` 文件夹中的所有文件上传
4. 在提交信息中输入：`初始部署：智缘益慷企业官网`
5. 点击 "Commit changes"

#### 方法二：通过Git命令行（推荐）
```bash
# 1. 克隆您的仓库到本地
git clone https://github.com/mazhaokang76-rgb/zhiyuan-yikang-website.git
cd zhiyuan-yikang-website

# 2. 将部署包中的所有文件复制到仓库目录
# (将 /workspace/GitHub部署包/ 下的所有内容复制到当前目录)

# 3. 提交并推送
git add .
git commit -m "初始部署：智缘益慷企业官网"
git push origin main
```

## 第二步：选择部署平台

### 方案一：Vercel部署（最简单，推荐）

1. **连接GitHub**
   - 访问：https://vercel.com
   - 点击 "Sign up with GitHub"
   - 授权访问您的GitHub仓库

2. **导入项目**
   - 点击 "New Project"
   - 选择您的仓库 `zhiyuan-yikang-website`
   - 点击 "Import"

3. **配置构建设置**
   - Framework Preset: `Vite`
   - Build Command: `npm run build` (自动检测)
   - Output Directory: `dist` (自动检测)
   - Install Command: `npm install` (自动检测)

4. **部署**
   - 点击 "Deploy"
   - 等待构建完成（通常2-3分钟）
   - 获得部署链接，例如：`https://zhiyuan-yikang-website.vercel.app`

### 方案二：Netlify部署

1. **连接GitHub**
   - 访问：https://netlify.com
   - 点击 "Sign up with GitHub"
   - 选择您的仓库

2. **构建设置**
   - Build command: `npm run build`
   - Publish directory: `dist`

3. **部署**
   - 点击 "Deploy site"
   - 获得部署链接

### 方案三：GitHub Pages（免费，但需要一些配置）

1. **启用GitHub Pages**
   - 进入仓库设置页面
   - 找到 "Pages" 设置
   - Source选择 "GitHub Actions"

2. **CI/CD自动部署**
   - 推送代码后，GitHub Actions会自动运行
   - 在 Actions 标签页查看部署状态
   - 部署完成后，访问 `https://mazhaokang76-rgb.github.io/zhiyuan-yikang-website`

## 第三步：配置自定义域名（可选）

如果您有自定义域名，可以配置CNAME记录指向部署地址。

### Vercel配置自定义域名：
1. 在Vercel项目设置中点击 "Domains"
2. 添加您的域名
3. 按照提示配置DNS记录

### Netlify配置自定义域名：
1. 在Netlify项目设置中点击 "Domain management"
2. 添加您的域名
3. 按照提示配置DNS记录

## 第四步：功能配置（可选）

如果需要启用Supabase后端功能：

### 4.1 环境变量配置
在部署平台的环境变量设置中添加：
```
VITE_SUPABASE_URL=您的Supabase项目URL
VITE_SUPABASE_ANON_KEY=您的Supabase匿名密钥
```

### 4.2 Supabase项目设置
如果您需要完整的后端功能，可以：
1. 注册Supabase账号：https://supabase.com
2. 创建新项目
3. 获取项目URL和API Key
4. 在部署平台配置环境变量

## 故障排除

### 常见问题：

1. **构建失败**
   - 检查package.json中的依赖是否正确
   - 确保所有文件都已正确上传

2. **页面空白**
   - 检查浏览器控制台是否有错误
   - 确认dist文件夹中包含正确的文件

3. **样式丢失**
   - 确保CSS文件路径正确
   - 检查vite.config.ts配置

4. **图片不显示**
   - 确认图片文件在public/images/目录中
   - 检查文件名大小写是否匹配

## 建议的部署流程

1. **测试部署**：首先使用Vercel快速部署，确保功能正常
2. **域名配置**：添加自定义域名
3. **内容更新**：根据需要更新网站内容
4. **性能优化**：配置CDN和缓存

## 技术支持

如果在部署过程中遇到问题，请检查：
1. GitHub仓库是否包含所有必需文件
2. 构建日志是否有错误信息
3. 浏览器控制台是否有JavaScript错误
4. 网络连接是否正常
